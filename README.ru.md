# JTF — JSON Token Format

[English](README.md) · **Русский**

Кодирование данных модели JSON без потерь и с обратимостью, которое сокращает
число токенов, расходуемых LLM на структурированные данные, — **−33 % к компактному
JSON** (замерено токенайзером `tiktoken` `cl100k_base`).

</div>

---

JTF — это не формат сжатия. На выходе получается читаемый человеком текст в UTF-8,
который языковая модель воспринимает и анализирует напрямую. Выигрыш достигается
за счёт отказа от избыточной пунктуации JSON, превращения массивов однотипных
записей в компактную таблицу и вынесения повторяющихся строковых значений —
при гарантии `decode(encode(x)) == x` для любого значения JSON.

Этот репозиторий превращает JTF в полноценный **стандарт**:

- **[SPEC.md](SPEC.md)** — авторитетная спецификация, не зависящая от реализации.
  По ней одной можно написать совместимый кодировщик и декодировщик.
- **[conformance/](conformance/)** — независимые от языка эталонные векторы,
  которые *задают* совместимость. Обе библиотеки проходят их полностью.
- **[python/](python/)** — эталонный пакет Python (`jtf`): encode, decode, CLI.
- **[js/](js/)** — модуль JavaScript (ESM): encode и decode.

## Как это выглядит

JSON:

```json
[
  {"id": 1, "name": "Alice", "active": true},
  {"id": 2, "name": "Bob",   "active": false}
]
```

JTF:

```
#2 id name active
	1	Alice	true
	2	Bob	false
```

Заголовок называет столбцы один раз; каждая строка — это просто ячейки через
табуляцию. Вложенные записи сворачиваются в точечные пути (`addr.city`), а
повторяющиеся строковые значения выносятся в небольшой словарь `$N`, когда это
экономит токены. Все правила описаны в [SPEC.md](SPEC.md).

## Установка и использование

### Python

```sh
pip install -e python/            # или: pip install -e "python/[tiktoken]"
```

```python
from jtf import encode, decode

text = encode({"id": 1, "name": "Ada"})
obj  = decode(text)               # == {"id": 1, "name": "Ada"}
```

CLI:

```sh
jtf encode data.json -o data.jtf
jtf decode data.jtf  -o roundtrip.json
jtf encode data.json --tiktoken   # подстроить словарь под реальный токенайзер
```

### JavaScript (Node ≥ 18, ESM)

```js
import { encode, decode } from "./js/src/jtf.js";

const text = encode({ id: 1, name: "Ada" });
const obj  = decode(text);        // глубоко равно { id: 1, name: "Ada" }
```

CLI:

```sh
node js/bin/jtf.js encode data.json -o data.jtf
node js/bin/jtf.js decode data.jtf  -o roundtrip.json
```

## Идея конформанса

Формат становится *стандартом* только тогда, когда независимые реализации
совпадают байт в байт. В каталоге `conformance/` хранятся входные файлы
`NNN-name.json` в паре с ожидаемым выводом `NNN-name.jtf` (индекс — в
`manifest.json`). Для каждого вектора совместимая библиотека обязана: (1) заново
закодировать вход ровно в ожидаемый `.jtf` и (2) декодировать `.jtf` обратно
ровно во вход. Источник истины — Python; JavaScript выровнен по нему; векторы —
это контракт.

./run-conformance.sh

## Результат по токенам (замер)

`tiktoken` `cl100k_base`, JTF (рабочий режим) против компактного JSON,
на шести эталонных документах:

| файл | JSON компактный | JTF | экономия |
|------|----------------:|----:|---------:|
| config.json | 133 | 133 | 0,0 % |
| users_table.json | 294 | 214 | 27,2 % |
| api_response.json | 289 | 299 | −3,5 % |
| logs.json | 556 | 399 | 28,2 % |
| repeated_values.json | 794 | 364 | 54,2 % |
| nested_uniform.json | 759 | 473 | 37,7 % |
| **ИТОГО** | **2825** | **1882** | **33,4 %** |

JTF выгоднее всего для массивов однотипных записей, повторяющихся строковых
значений и вложенных записей; на небольших документах со сплошь уникальными
значениями возможен небольшой проигрыш. Цифра воспроизведена из исходного
проекта, а не выдумана заново.

## Дорожная карта

- **Порт на Rust** (`rust/`) — запланирован, проверяется теми же векторами.
- **Порт на Go** (`go/`) — запланирован, тот же контракт.
- Потоковые encode/decode для очень больших массивов.

Порты пока не реализованы; их приёмочным тестом станут те же векторы из
`conformance/`, что и сегодня для Python и JS.

## Связанные проекты

- _JTF изначально вышел как **json-token-format** — теперь сведён сюда как каноничный дом (старые ссылки редиректят)._
- **token-diet** — интерактивная площадка, показывающая экономию JTF на живых
  токенайзерах GPT.
- **llmcost** — инструменты подсчёта токенов и стоимости для промптов LLM.
- **llm-gateway** — токен-экономный LLM-прокси, сжимающий полезную нагрузку через JTF.
